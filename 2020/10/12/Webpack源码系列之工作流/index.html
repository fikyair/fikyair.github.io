Compilation 类

声明一个 Compilation.js 类，主要是执行编译工作
编译模块思路：

  1. 从硬盘上把模块内容读出来，读成一个文本
  2. 可能不是一个 JS 模块，所以可能会要走 loader 的转换，最终肯定要得到一个 JS 模块代码，得不到就报错
  3. 把这个 JS 模块代码经过 parser（webpack 内部是 acorn ） 处理转成抽象语法树 AST
  4. 分析 AST 里面的依赖，也就是找 require import 节点，分析依赖的模块
  5. 递归的编译依赖的模块
  6. 不停的一次递归执行上面 5 步，直到所有模块都编译完成为止

通过 NormalModuleFactory 新增 NormalModule 编译模块和依赖，然后转换成 AST 语法树，放入 ast set 中

Webpack 的插件机制

Webpack 本质上是一种事件流的机制，它的工作流程就是将各个插件串联起来，而实现这一切的核心就是 Tapable ，Webpack 中最核心的负责编译的 Compiler 和负责创建 bundle 的 Compilation 都是 Tapable 的实例

Webpack 的 Tapable 事件流机制保证了插件的有序性，将各个插件串联起来， Webpack 在运行过程中会广播事件，插件只需要监听它所关心的事件，就能加入到这条 Webpack 机制中，去改变 Webpack 的运作，使得整个系统扩展性良好。

Tapable 也是一个小型的 library，是 Webpack 的一个核心工具。类似于 node 中的 events 库，核心原理就是一个订阅发布模式。作用是提供类似的插件接口

tapable 如何将 webpack 插件关联 ?

Tapable 类暴露了 tap、tapAsync 和 tapPromise，用来注册事件

使用 call，callAsync 和 promise 来触发事件

webpack 实现插件机制的大体方式是：

  • 创建 - Webpack 在其内部对象 Compiler 上创建各种钩子
  • 注册 - 插件将自己的方法注册到对应钩子上，交给 Webpack
  • 调用 - Webpack 编译过程中，会适时地触发相应钩子，因此也就触发了插件的方法
    通过事件和注册和监听，触发 Webpack 生命周期中的函数方法

tapable 分类

Hook 类型可以分为 同步Sync 和 异步Async，异步又分为并行和串行
